## Project Overview

A Nuxt 4 chat application (Vue 3, Nuxt UI 4, Tailwind CSS 4) that talks to LLMs through the Vercel AI SDK. Package manager is **Bun**. Chat data is currently mock/in-memory only — there is no database or persistence layer yet.

## Common Commands

```bash
bun dev          # Dev server at http://localhost:3000
bun build        # Production build
bun preview      # Preview production build
bun generate     # Static site generation
bun lint         # ESLint
bun lint:fix     # Auto-fix ESLint issues
```

ESLint is auto-generated by `@nuxt/eslint` (`eslint.config.mjs` wraps `.nuxt/eslint.config.mjs`). A Husky pre-commit hook runs `bun lint` before every commit.

To exercise the AI endpoint you must provide an OpenRouter key. Runtime config keys are confusingly named `openai*` but the provider is **OpenRouter**. Override at runtime via env vars:

```bash
NUXT_OPENAI_API_KEY=sk-or-...  NUXT_OPENAI_MODEL=gpt-4o-mini  bun dev
```

## Architecture

### Nuxt Layers (most important to understand)

The app is split into [Nuxt layers](https://nuxt.com/docs/getting-started/layers) under `layers/`. Nuxt **auto-extends** every directory in `layers/` — there is no explicit `extends` array in the root `nuxt.config.ts`. Each layer has its own `nuxt.config.ts`, and Nuxt deep-merges them with the root config.

- **`layers/base/`** — shared shell: `default.vue` layout, `AppHeader`, `AppSidebar`, `MarkdownRenderer`, global CSS, theme `app.config.ts`, and the `@nuxt/ui` + `@nuxtjs/mdc` module registration (incl. the `tokyo-night` code-highlight theme).
- **`layers/chat/`** — the chat feature: pages, `useChat*`/`useProject*`/`useSidebar` composables, mock data, and the server API/services. Defines the `runtimeConfig` (`openaiApiKey`, `openaiModel`).
- **Root `app/`** — thin entry: `app.vue` (wraps everything in `<UApp>` + `<NuxtLayout>`/`<NuxtPage>`) and `types.ts` (shared `Chat`, `ChatMessage`, `Project` types, imported as `~/types`).

**Consequence:** config is spread across `nuxt.config.ts` (root + each layer) and `app.config.ts` (root + base layer) — they all merge. When changing modules, CSS, runtime config, or UI theme, find the right layer's config rather than assuming the root file.

### Routing

File-based routes live in `layers/chat/app/pages/`:
- `index.vue` → `/`
- `chats/[id].vue` → `/chats/:id`
- `projects/[projectId]/index.vue` → `/projects/:projectId`
- `projects/[projectId]/chats/[id].vue` → `/projects/:projectId/chats/:id`

### State

Client state is held in `useState`-backed composables (SSR-shared singletons), not a store library:
- `useChats` — chat list + `createChat`/`createChatAndNavigate`/`chatsInProject` (seeded from `mockData.ts`).
- `useChat(chatId)` — single chat view; `sendMessage` POSTs the full message array to `/api/ai` and appends the assistant reply.
- `useSidebar`, `useProjects`, `useProject` — sidebar toggle and project state.

### Server

- `server/api/ai.ts` — POST endpoint; builds an OpenRouter model from runtime config and calls `generateChatResponse`.
- `server/services/ai.service.ts` — wraps the AI SDK (`createOpR`, `generateChatResponse` via `generateText`).
- `server/utils/validation.ts` — `pipeBody(schema, event)` validates request bodies with a Zod schema and throws a 400 on failure. Use this for new endpoints; schemas go in `server/schemas/`.

### Tech Stack

Nuxt 4.2 · Vue 3 · @nuxt/ui 4 · Tailwind CSS 4 · `ai` SDK 6 + `@openrouter/ai-sdk-provider` · Zod 4 · `@nuxtjs/mdc` for markdown rendering · Node v24 (`.nvmrc`).
